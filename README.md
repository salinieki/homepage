# Salinieki Homepage

Hugo-based website for the Latvian folk dance group **Salinieki** in London — [salinieki.co.uk](https://salinieki.co.uk/).

## Development setup (macOS)

1. Install [Homebrew](https://brew.sh/) if it is not already installed.
2. Install Hugo (the extended build is required so SCSS / asset processing work):

   ```sh
   brew install hugo
   ```

3. Clone the repository and `cd` into it:

   ```sh
   git clone <repo-url> salinieki
   cd salinieki
   ```

4. Start the local development server:

   ```sh
   hugo server -D
   ```

   The site is then available at <http://localhost:1313/>. Hugo watches files and reloads the browser automatically when you save changes.

To produce a production build into `public/`:

```sh
hugo --minify
```

## Adding a news article about a concert

A concert appears in several places on the site. To add one, do the following steps in order.

### 1. Create the article file

Add a new Markdown file under `content/news/`, named `YYYY-MM-DD-short-slug.md` (the date should be the date of the article). Use an existing article as a template, for example `content/news/2026-05-05-lestere-kulturas-diena.md`.

The front matter should look like this:

```yaml
---
title: Concert title
date: 2026-06-15
location: City, Country
coverImage: images/foto/2026-06-event-slug/01-cover.jpg
summary: >
  One or two sentences shown on the news list.
hasFullArticle: true
---
```

- `coverImage` is the thumbnail shown on the home page news list.
- `hasFullArticle: true` makes the title link to the full article page and enables previous/next navigation between articles. Omit it (or set to `false`) if you only want a short news entry without a dedicated page.

The body of the file is normal Markdown. To include photos and embedded videos in the article, use the shortcodes:

```markdown
{{< figure src="images/foto/2026-06-event-slug/photo.jpg" alt="Description" >}}

{{< youtube id="YOUTUBE_VIDEO_ID" title="Video title" >}}
```

### 2. Add the photos

Place the concert photos as JPG files in:

```
assets/images/foto/YYYY-MM-short-slug/
```

Use the same `<event-slug>` you used in the `coverImage` path (e.g. `2026-06-event-slug`). Hugo automatically generates resized thumbnails for galleries from this folder. Number the files (`01-…`, `02-…`, etc.) to control the order they appear in.

### 3. Add a photo gallery on the Foto page

Edit `content/foto.md` and add a new section at the top:

```markdown
### 2026 - Concert title

{{< gallery id="2026-06-event-slug" alt="Concert title" >}}
```

The `id` must match the folder name under `assets/images/foto/`. The shortcode picks up every `.jpg` in that folder automatically.

### 4. Add the concert to the list on Par mums

Edit `content/par-mums.md` and add a new `<li>` at the top of the `<ul class="concert-list">` block, keeping the list in reverse-chronological order:

```html
<li><strong>2026. Jūnijs</strong><span>Concert title, City, Country</span></li>
```

### 5. (Optional) Update the hero slideshow

If a particularly good cover photo from the concert should rotate on the home page hero, add it to `[[params.heroSlides]]` in `hugo.toml`:

```toml
[[params.heroSlides]]
  image = "images/foto/2026-06-event-slug/01-cover.jpg"
```

## Project structure

A short tour of the top-level folders:

- **`content/`** — all the text content of the site, written as Markdown files. Each page on the site corresponds to one file here (e.g. `par-mums.md`, `kontakti.md`, the news posts in `news/`).
- **`assets/`** — image files (and other media) that Hugo processes before serving. Concert photos go in `assets/images/foto/<event-slug>/`. Hugo automatically resizes them for galleries and thumbnails.
- **`static/`** — files that are served as-is, without any processing. This contains the stylesheet (`static/css/style.css`), JavaScript (`static/js/site.js`), fonts, and the site logo.
- **`layouts/`** — the HTML templates that wrap your Markdown content. `baseof.html` is the outer page shell; `index.html` is the home page; `_default/single.html` is used for individual article pages; `partials/` contains reusable pieces like the header, footer, sidebar, and hero; `shortcodes/` contains the `{{< gallery >}}` and `{{< figure >}}` helpers used inside Markdown.
- **`archetypes/`** — templates used by `hugo new` to scaffold new content files.
- **`data/`** and **`i18n/`** — Hugo's standard folders for structured data and translations. Currently empty in this project.
- **`themes/`** — would hold third-party Hugo themes. Empty here, because the site uses its own custom layouts.
- **`resources/`** — auto-generated cache of processed images (resized thumbnails, etc.). You don't usually edit this directly; Hugo manages it.
- **`public/`** — the built site, generated by `hugo --minify`. This is what gets deployed. Safe to delete; it will be regenerated on the next build.
- **`hugo.toml`** — site-wide configuration: title, menu items, hero slides, calendar entries, supporters, social links.
